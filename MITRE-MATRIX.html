Healthcare threat model: The primary adversary is ransomware groups targeting EHR systems, clinical workstations, and patient data. Secondary is nation-state espionage targeting research and pharmaceutical IP. Flat hospital networks make lateral movement trivial once initial access is established. Cloud EHR adoption (Epic, Cerner) increases OAuth/identity surface dramatically.
Priority Ecosystem Focused Rules for Healthcare Healthcare-Specific Context Key Techniques
P1 CRITICAL
Ransomware Cousins
  • Cousin_EncryptionVelocity
  • Cousin_ShadowDeletion
  • Cousin_ServiceStopPlus
 Healthcare is #1 ransomware target globally. Patient safety impact (cancelled operations, diverted ambulances). EHR downtime = direct clinical risk. Shadow copy deletion is the last line before encryption. Detect here or it's too late. T1486 T1490 T1489
P1 CRITICAL
Credential Access
  • COMPOSITE_L3_LSASS
  • L2_Composite_NTDS_Dit
  • SecretsRecon_Hunt
  • Cousin_Discovery_Logon
 Clinical staff share credentials on shared terminals. LSASS dump on a clinical workstation = immediate domain escalation. NTDS dump = entire trust model gone. Healthcare AD forests are often under-hardened with legacy systems preventing Credential Guard. T1003.001 T1003.003 T1552
P1 CRITICAL
SMB Lateral Movement
  • SMB_Services_PsExec
  • SMB_TaskExec_Svchosts
  • ADX_SMB_Svhosts_Harvester
 Hospital networks are notoriously flat. Clinical systems, PACS servers, infusion pumps, and domain controllers often share the same network segment. SMB lateral movement from one clinical workstation can reach imaging, pharmacy, and theatre systems within minutes. T1021.002 T1569.002 T1543.003
P2 HIGH
OAuth / Identity
  • OAuth_Consent_Abuse
  • OAuth_Token_Theft
  • Service_Principal_Backdoor
 Cloud EHR migration (Epic on Azure, Cerner cloud) dramatically expanded OAuth attack surface. Clinician M365 accounts have access to patient data. Consent abuse can grant persistent access to patient records without touching endpoints. Service principal backdoors survive credential resets. T1528 T1550.001 T1078.004
P2 HIGH
WMI Ecosystem
  • WMI_FILELESS_ATTACK
  • WMI_Permanent_Subscription
  • WmiPrvSE_Secondary
 WMI permanent event subscriptions are the preferred persistence method in healthcare environments because AV coverage is often limited to signature detection. Clinical workstations run Windows 7-10 with minimal EDR. WMI fileless survives reimaging if the subscription survives. T1047 T1546.003 T1059.001
P2 HIGH
C2 Ecosystem
  • C2_HTTPS_Jitter_Beacon (v2)
  • C2_NamedPipe_IPC_Session
 Healthcare attackers maintain long dwell times — average 200+ days before ransomware deployment. HTTPS beaconing blends with legitimate EHR traffic to cloud vendors. Named pipe C2 (Cobalt Strike) is the preferred post-exploitation channel in healthcare breaches. Jitter detection is essential against modern C2 frameworks. T1071.001 T1559.001 T1102
P3 MEDIUM
LOLBins HuntPack
  • 01_ProxyExec_Ingress
  • Powershell_Cradle_Ingress
  • PowerShell_Download
  • LOLBINS_Ingress_Pack
 Clinical workstations run legacy software requiring LOLBin execution for legitimate tasks — making LOLBin detection harder. Focus on ingress and cradle detection over execution detection to avoid false positive flood. Healthcare IT teams use PowerShell extensively for patch management. T1218 T1059.001 T1105

Healthcare-Specific Missing Coverage
PHI Exfiltration Detection P1
No dedicated staging + archive + large outbound transfer chain. Patient data exfil before ransomware is now standard (double extortion). Need: file archive creation → staging folder → outbound transfer composite.
RDP Clinical Workstation Abuse P1
No RDP lateral movement ecosystem. Hospital environments run extensive RDP for remote clinical support. RDP is the #1 initial access vector in healthcare. Need: T1021.001 ecosystem with session hijack cousin rules.
Backup System Targeting P1
Shadow deletion cousin exists but no dedicated backup software process kill (Veeam, Backup Exec, Windows Server Backup). Pre-ransomware backup destruction is systematic. Need dedicated T1490 composite.
VPN / Remote Access Abuse P2
Post-COVID remote clinical access explosion. No detection for impossible travel, concurrent VPN sessions, off-hours VPN from unusual geos. T1133 External Remote Services gap.
Medical Device Network Pivot P2
IoMT (infusion pumps, imaging, monitoring equipment) runs unpatched firmware on flat networks. No detection for pivot from medical device subnet to clinical workstation domain. Unique to healthcare.
HL7 / FHIR API Abuse P2
Cloud EHR APIs (Epic FHIR R4, Cerner Millennium) are OAuth protected but bulk patient data access through legitimate API tokens is undetected. Need anomaly detection on API query volume and data scope.
Coverage gap philosophy: Missing rules are identified by ecosystem shape, not MITRE square count. A gap exists where the attack intent is covered on one vector but the cousin surfaces are not mapped. These are the paths an adversary pivots to when the primary detection fires.
Priority 1 — Missing Entire Ecosystems
Exfiltration Ecosystem
No dedicated exfiltration detection. Data staging → archive creation → outbound transfer composite is absent. T1041, T1048, T1567. Critical for financial and healthcare. This is the ecosystem where the damage is actually measured.
Initial Access Ecosystem
No phishing chain detection (T1566), no drive-by compromise (T1189), no supply chain (T1195). Everything in the framework assumes post-initial-access. The entry point ecosystem is entirely missing.
Kerberos Attack Ecosystem
Kerberoasting (T1558.003), AS-REP Roasting (T1558.004), Pass-the-Ticket (T1550.003), Golden Ticket (T1558.001) — zero coverage. These are the primary AD privilege escalation chain in enterprise environments.
Cloud Resource Abuse
Beyond OAuth — no Azure/AWS resource creation (T1578), no compute hijack for cryptomining (T1496), no S3/Blob storage data exfil (T1530). Cloud coverage is identity-only currently.
DNS Tunnelling C2 Ecosystem
HTTPS beacon exists. DNS C2 (T1071.004) is absent as a dedicated ecosystem. DNS tunnelling is increasingly common as a backup C2 channel when HTTPS is blocked or detected.
DCOM Lateral Movement
SMB and WMI lateral covered. DCOM (T1021.003) is the third primary lateral movement path — MMC20, ShellWindows, ShellBrowserWindow. No cousin rules mapped for this surface.
RDP Ecosystem
T1021.001 completely absent. RDP is #1 lateral movement vector in enterprise environments. Session hijack, sticky keys backdoor, RDP tunnel, RDP to cloud — all unmapped cousin surfaces.
Email-Based Threat Chain
Given Mimecast/Cofense background — email threat chain detection would be a natural extension. Attachment execution chains, macro execution, HTML smuggling — none present in current framework.
Priority 2 — Missing Cousin Rules Within Existing Ecosystems
C2 → DNS Tunnelling Cousin
HTTPS and Named Pipe C2 covered. DNS tunnelling (iodine, dnscat2) is the natural cousin — same intent, different protocol. Single rule needed to close the ecosystem.
Credential Access → Kerberoasting Cousin
LSASS and NTDS covered. Kerberoasting uses the legitimate Kerberos TGS mechanism — no LSASS touch. Different substrate, same credential theft intent. Classic cousin gap.
Lateral → WinRM Cousin
SMB and WMI lateral covered. WinRM (T1021.006) is the third leg — same intent, PSRemoting substrate. Evil-WinRM is standard post-exploitation tooling.
Persistence → COM Hijack Cousin
Registry and ScheduledTask persistence covered. COM object hijack (T1546.015) is user-land persistent without registry run key writes — different substrate, same persistence intent.
Impact → Network Share Encryption
Local encryption velocity covered. Ransomware increasingly encrypts network shares before local files. SMB write velocity to UNC paths is the missing cousin to local encryption detection.
Identity → Device Code Phishing
OAuth consent abuse covered. Device code phishing (T1528 variant) uses legitimate OAuth device auth flow — different substrate, same token theft intent. Increasingly used against M365.
