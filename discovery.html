🔵 BLUE TEAM
Individual discovery commands are completely normal. The composite anchors on velocity and chaining: 5+ distinct discovery commands executed by the same process within 120 seconds. This is structurally a script or automated tool — not a human admin.

Discovery command taxonomy: whoami, net user, net group, ipconfig, arp -a, netstat -an, systeminfo, tasklist, net share, net view. Any 5+ from this set within 2 minutes = post-exploitation enumeration.
🔴 RED TEAM
Standard post-exploitation runbook (Cobalt Strike/Empire): Automated situational awareness executes within seconds of beacon establishment: whoami /all → net user /domain → net group "Domain Admins" /domain → ipconfig /all → arp -a → netstat -an → systeminfo.

This burst pattern is identical across Cobalt Strike, Metasploit, Empire, and manual pentests — it is the fingerprint of post-exploitation enumeration regardless of tool.

BloodHound AD Enumeration via LDAP
T1069.002 · T1087.002
▼
📎 INSERT RULE LINK 🔗 T1069.002 🔗 BloodHound
⚡ KILL CHAIN: Discovery → Lateral Movement path planning
🔵 BLUE TEAM
BloodHound ingests massive LDAP query volumes to map AD attack paths. Detection: anomalous LDAP query volume from a workstation (not a DC or management server), queries for sensitive attributes (msds-allowedtodelegateto, AdminCount, SIDHistory, ms-DS-MachineAccountQuota) in rapid succession, SAMR enumeration of domain groups from non-admin accounts. Threshold: >500 LDAP queries from a single workstation in 5 minutes.
🔴 RED TEAM
BloodHound is the attacker's GPS for AD. Running SharpHound collector takes 2-3 minutes and maps every attack path to Domain Admin — which users are local admins on which machines, which machines have unconstrained delegation, Kerberoastable accounts with crack priority, shortest path to DA. Transforms lateral movement from guesswork to shortest-path algorithm. Most enterprise compromises that reach Domain Admin used BloodHound or equivalent.

Internal Network Port Scan
T1046
▼
📎 INSERT RULE LINK 🔗 T1046 Network Service Discovery
⚡ KILL CHAIN: Discovery → Lateral Movement target identification
🔵 BLUE TEAM
Internal port scanning from a compromised host: a workstation making TCP connections to 10+ distinct internal hosts on the same ports within a 5-minute window. Key ports: 445 (SMB), 3389 (RDP), 5985 (WinRM), 1433 (SQL), 22 (SSH), 80/443 (web). Scanning these ports internally identifies targets for lateral movement substrates — the discovery directly feeds the lateral movement cousin cluster.
🔴 RED TEAM
Post-exploitation network sweep to identify live hosts and open services. Tools: nmap -sn 192.168.1.0/24 via Cobalt Strike SOCKS proxy, CrackMapExec for SMB/WinRM sweep, or simple PowerShell: 1..254 | % {Test-Connection -Count 1 192.168.1.$_ -AsJob}. Identifies domain controllers (port 88/389), file servers (445), admin workstations (3389) — prioritisation for lateral movement.
