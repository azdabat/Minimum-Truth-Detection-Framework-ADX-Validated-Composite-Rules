📎 INSERT RULE LINK · KQL · Sentinel Network Analytics 🔗 T1071.001 🔗 T1573 Encrypted Channel 📄 See Novel Research: EtherRAT Blockchain C2
🔵 BLUE TEAM
C2 beaconing is structurally distinct from human web traffic: it is periodic, consistent in payload size, and persists regardless of user activity (including overnight when no human is at the keyboard). Statistical composite: standard deviation of connection intervals to a single external host below threshold (machine-driven regularity), consistent byte sizes across connections, persistence overnight or across weekends when the user is absent.

Jitter is accommodated by analysing the distribution across 24 hours rather than exact intervals. Reinforcement: new domain (<30 days), no HTTP Referer header, anomalous JA3/JA3S hash, parent process not a browser.
🔴 RED TEAM
Cobalt Strike default profile: 60-second sleep, 0% jitter. Trivially caught by statistical analysis. Mature operators configure 1-4 hour sleep with 25-40% jitter and malleable C2 profiles mimicking legitimate browser traffic — matching JA3 hashes of Chrome, mimicking Google Analytics or Akamai CDN request patterns.

Brute Ratel C4 and Sliver C2 are Cobalt Strike alternatives with built-in OPSEC profiles designed to evade statistical beaconing detection.

When all HTTP/S channels are blocked: EtherRAT uses the Ethereum blockchain as C2 — completely uncensorable.

DNS Tunnelling C2
T1071.004
▼
📎 INSERT RULE LINK 🔗 T1071.004 DNS
⚡ KILL CHAIN: C2 (firewall-bypass — port 53 UDP almost never blocked)
🔵 BLUE TEAM
DNS tunnelling encodes C2 commands in DNS query subdomain labels and responses in TXT records. Detection signals: anomalous DNS query length (legitimate queries rarely exceed 50 chars in subdomain), high entropy in subdomain labels (base64/hex encoded data), high query frequency to a single domain (beacon polling), TXT record responses with unusually long or high-entropy content. Monitor for DNS queries to newly registered domains or domains without corresponding HTTP/S traffic.
🔴 RED TEAM
Why DNS: Port 53 UDP is almost never blocked outbound. Many organisations don't log DNS at all — blind spot. The attacker controls the authoritative DNS server for their domain — every DNS query is a C2 check-in, every response is a command.

Command in query: aGVsbG8=.cmd.evil.com — base64-encoded command in subdomain label. Response in TXT record: base64-encoded output.

C2 via Legitimate Web Services
T1102 · T1102.002
▼
📎 INSERT RULE LINK 🔗 T1102 Web Service
⚡ KILL CHAIN: C2 (blend into legitimate SaaS traffic — impossible to block by domain)
🔵 BLUE TEAM
C2 through legitimate services (GitHub, Slack, Teams, OneDrive, Pastebin, Twitter/X) uses trusted infrastructure that cannot be blocked without disrupting business. Detection:** focus on the process making the connection rather than the destination domain. A legitimate business process (Word, Excel, scheduled task result) connecting to GitHub API or Slack API is anomalous — the endpoint is legitimate but the initiating process context is not.
🔴 RED TEAM
GitHub C2: Operator posts commands in a GitHub Gist or repository. Implant polls the Gist every N minutes, reads new commits, executes commands, writes output to another Gist. All traffic is to api.github.com — a domain no organisation can block. Same pattern works with Slack (webhook C2), OneDrive (SharePoint API), and Teams channels.

Used by APT29 (NOBELIUM) — C2 through legitimate Microsoft infrastructure.

Blockchain C2 — EtherRAT / Immutable Channel
T1071 · Novel
▼
📎 INSERT RULE LINK · React2Shell KQL 📄 Full Novel Research: EtherRAT / React2Shell CVE-2025-55182
⚡ KILL CHAIN: C2 (completely uncensorable — no C2 server to sinkhole)
🔵 BLUE TEAM
Blockchain C2 embeds commands in Ethereum transaction input data. The RAT polls a public blockchain API (Infura, Alchemy) rather than a dedicated C2 server. Detection: outbound connections to Ethereum RPC endpoints (mainnet.infura.io, eth-mainnet.alchemyapi.io) from non-developer hosts, regular polling intervals to blockchain API endpoints, processes parsing transaction input field data and executing the result. See MTDF Novel Research for the full detection suite.
🔴 RED TEAM
Why blockchain C2 is a step change: There is no C2 server to sinkhole, no domain to block, no IP to blacklist. The C2 infrastructure IS the Ethereum blockchain — a public, distributed, censorship-resistant ledger. Commands delivered by sending an Ethereum transaction (cost: pennies) with encoded command in the input data field. Entire C2 channel operates over legitimate HTTPS to blockchain infrastructure providers.

Domain Fronting via CDN
T1090.004
▼
📎 INSERT RULE LINK 🔗 T1090.004 Domain Fronting
⚡ KILL CHAIN: C2 (masks true C2 destination behind legitimate CDN domain)
🔵 BLUE TEAM
Domain fronting uses a CDN's infrastructure so that the TLS SNI (visible in network inspection) shows a trusted domain (e.g. microsoft.com via Azure CDN) while the actual HTTP Host header inside the encrypted tunnel routes to the attacker's C2. Detection: TLS SNI / HTTP Host header mismatch, JA3 hash analysis for known Cobalt Strike/Brute Ratel profiles, anomalous certificate subject vs. connection destination.
🔴 RED TEAM
Domain fronting was heavily used by APT29 (NOBELIUM) via Azure CDN — all traffic appeared destined for Microsoft infrastructure. Most major CDN providers (AWS CloudFront, Azure CDN, Cloudflare) have now implemented controls to prevent domain fronting, but misconfigured CDN tenants may still be abusable.
