🔵 CORE BASELINE COMPOSITE
Run Key Write — Non-System Process + Anomalous Executable Path Anchor
📎 INSERT RULE LINK · KQL · Sentinel + MDE 🔗 MITRE T1547.001
🔵 BLUE TEAM
Run key writes are noisy — legitimate software uses them constantly. The composite anchors on path anomaly: run key values pointing to executables in TEMP, AppData, ProgramData, or user-writable paths. Key insight: combine the registry write event with the writing process. If a user-space process (Word, browser, non-IT tool) writes to HKCU/HKLM Run, that is structurally suspicious regardless of payload.

Reinforcement: the pointed-to executable was created <60 seconds before the registry write, or has a double extension (.pdf.exe), or the run key name mimics a legitimate software entry.
🔴 RED TEAM
Run keys are commodity-malware standard. Used by virtually every RAT, banking trojan, and ransomware dropper as primary or backup persistence.

Chain: Dropper lands payload in AppData → writes HKCU\Software\Microsoft\Windows\CurrentVersion\Run\[random] = C:\Users\[user]\AppData\Roaming\[random].exe → persists across reboots without admin rights.

Evasion: naming the key after legitimate software (AdobeUpdate, ChromeHelper), using environment variable expansion (%APPDATA%\payload.exe) to obscure the full path.

DLL Search Order Hijacking / Sideloading
T1574.001 · T1574.002
▼
📎 INSERT RULE LINK 🔗 T1574.002 DLL Sideloading
⚡ KILL CHAIN: Persistence → Defense Evasion (executes via legitimate signed app)
🔵 BLUE TEAM
DLL sideloading places a malicious DLL in a directory where a legitimate application will load it in preference to the real system DLL. Minimum truth: a DLL with a known-system-DLL name appearing in a non-system path, loaded by a legitimate signed application. This cousin is also structurally linked to BYOVD — same substrate (DLL loading) but different intent vector (persistence via application trust rather than kernel exploitation).
🔴 RED TEAM
Classic chain: Drop malicious version.dll next to a signed Microsoft binary that loads version.dll (e.g. OneDriveSetup.exe). When the binary runs on startup → loads malicious DLL → DLL proxies all exports to real version.dll while executing the payload in a new thread. Application works normally. No registry modification. No scheduled task.

Extremely popular with Chinese APT groups — PlugX and ShadowPad both use DLL sideloading as primary persistence.

COM Object Hijacking — HKCU Registry
T1546.015
▼
📎 INSERT RULE LINK 🔗 T1546.015
⚡ KILL CHAIN: Persistence (no admin required, UAC bypass capable)
🔵 BLUE TEAM
COM hijacking writes a CLSID entry in HKCU (user-level, no admin required) that overrides the system HKLM entry. When a legitimate application loads that COM object, it loads the attacker's DLL instead. Detection: new CLSID entries in HKCU\Software\Classes\CLSID pointing to non-standard DLL paths, especially for CLSIDs known to be loaded by commonly executed applications (Explorer, Task Scheduler).
🔴 RED TEAM
Why COM hijacking: Requires no admin rights (HKCU writable by any user), survives reboots, appears as legitimate COM object loading by a trusted application, and can achieve UAC bypass when the hijacked COM object is used by an auto-elevating binary. Used in targeted attacks where the attacker wants to blend into normal Windows behaviour.

Malicious Service Installation
T1543.003
▼
📎 INSERT RULE LINK 🔗 T1543.003
⚡ KILL CHAIN: Persistence + Privilege Escalation (runs as SYSTEM)
🔵 BLUE TEAM
Event 7045 (new service installed) combined with binary path pointing to a non-standard location is a high-fidelity signal. Composite: 7045 where the service binary path is in TEMP, AppData, or ProgramData, OR service name is random characters, OR service is deleted within 120 seconds of creation (PsExec-style transient service). Services run as SYSTEM by default — any malicious service achieves immediate privilege escalation.
🔴 RED TEAM
Used for both persistence (start on boot) and execution (immediate SYSTEM shell). PsExec-style lateral movement uses transient services — installed, used, deleted. For persistence: install service pointing to attacker binary with start= auto — executes as SYSTEM on every system boot without user interaction.
