📤 EXFILTRATION — The objective of most financially-motivated intrusions. Minimum truth doctrine: unusual volume of outbound data to a non-corporate destination, or data leaving via a protocol not normally used for outbound transfer. The cousin doctrine maps all exfil substrates — block cloud storage exfil and the attacker uses DNS exfiltration. Block large transfers and they throttle to stay under volume thresholds. Detection must cover the intent: data leaving unauthorised.

🔵 BLUE TEAM
Volume-based exfil detection anchors on statistical deviation from baseline: a host transferring >3 standard deviations from its 30-day average outbound volume in a single session. Reinforcement: destination is a file-sharing or cloud storage domain (mega.nz, anonfiles, gofile.io, transfer.sh, file.io), destination domain is <30 days old, transfer occurs outside business hours, process making the transfer is not a known corporate sync client.

The 30-day baseline is critical — without it, volume thresholds produce excessive false positives for legitimate large transfers.
🔴 RED TEAM
Double-extortion exfil pattern (BlackCat/Cl0p/LockBit): Before encrypting, operators exfiltrate 50-500GB as proof and leverage. Tools: rclone syncs to cloud storage, MEGAcmd uploads directly to MEGA, custom HTTPS upload scripts.

Throttled exfil (evasion): Mature operators throttle transfer speed to stay under DLP volume thresholds — exfiltrating over 48-72 hours at 1-5 Mbps rather than a single high-volume burst. The per-hour volume stays below detection thresholds; only cumulative multi-day analysis reveals the pattern.

DNS Exfiltration — Data in Query Labels
T1048.003
▼
📎 INSERT RULE LINK 🔗 T1048.003
⚡ KILL CHAIN: Exfiltration (bypasses all HTTP/HTTPS DLP controls)
🔵 BLUE TEAM
Data encoded in DNS query subdomain labels — each query carries ~60 bytes of encoded data. Slow but impossible to block without breaking DNS resolution. Detection: anomalous DNS query volume to a single domain (>100 queries/minute), high entropy in subdomain labels, consistent query length across all queries to the same domain (automated encoding pattern), TXT record request volume anomaly. Requires DNS logging — many organisations log no DNS traffic at all.
🔴 RED TEAM
Use case: Air-gapped or heavily DLP-controlled environments where HTTPS is inspected. Every file can be encoded in base32 and transmitted as DNS queries — rate limited only by DNS resolver performance. DNScat2 supports both C2 and file transfer over DNS.

Exfiltration via Email / Webmail
T1048 · T1567
▼
📎 INSERT RULE LINK
⚡ KILL CHAIN: Exfiltration (blends into legitimate email traffic)
🔵 BLUE TEAM
Email exfiltration uses the corporate email infrastructure or external webmail against itself. Detection: anomalous outbound email volume from a single account (especially outside business hours), emails to personal accounts or free webmail providers with large attachments, emails to newly registered domains, Exchange/O365 email forwarding rules created to external addresses (T1114.003 — auto-forward to attacker-controlled mailbox).
🔴 RED TEAM
Auto-forward rule: With access to a compromised mailbox, create an inbox rule forwarding all incoming mail to an attacker-controlled address. The rule persists — even if the initial access is remediated, mail continues to be forwarded. Invisible to the victim. Used by APT29 against O365 environments for intelligence collection.
