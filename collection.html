🔵 CORE BASELINE COMPOSITE
Archive Creation in Non-Standard Path + Sensitive File Type Inclusion Anchor
📎 INSERT RULE LINK · KQL · MDE FileCreated events 🔗 T1560.001
🔵 BLUE TEAM
Attackers compress data before exfiltration to reduce transfer time and evade DLP controls. Minimum truth: archive utility (7zip, WinRAR, tar) creating archives in TEMP/ProgramData/network shares containing password-protected compressed files with sensitive file type inclusions (.docx, .xlsx, .pdf, .kdbx, .key, .pem, .pfx).

Correlate with subsequent outbound HTTPS/FTP connections from the same host within 60 minutes. Password-protected archives are a strong reinforcement signal — legitimate backup operations rarely use passwords.
🔴 RED TEAM
Standard double-extortion exfil prep:
7z a -p[password] staged.zip "C:\Users\*\Documents\*.docx" "C:\Users\*\Desktop\*.xlsx"

Password protection defeats content inspection by DLP tools. The archive is then uploaded to a cloud service (Dropbox, Google Drive, MEGA) — indistinguishable from legitimate traffic without DLP that can inspect the upload endpoint and volume.

Ransomware operators (BlackCat, Cl0p, LockBit) exfiltrate 50-500GB before encryption as leverage for double extortion.

⚡ KILL CHAIN: Collection (high-value intelligence — M&A, legal, executive comms)
🔵 BLUE TEAM
Email collection via Exchange or O365 Graph API provides high-value intelligence — M&A discussions, legal matters, executive communications, credentials in email body. Detection: anomalous EWS (Exchange Web Services) or Graph API mailbox access — large volume of email folder enumeration and message download by non-email-client user agents, mailbox access from unexpected source IPs or applications, Graph API Mail.Read permission exercise at volume.
🔴 RED TEAM
O365 email exfil with compromised credentials: MailSniper searches all mailboxes for keywords (password, VPN, secret) via EWS. Ruler automates Exchange interaction. With a Global Admin token, all employee mailboxes accessible via Graph API — often targeted for intelligence gathering before ransomware deployment.

Screen Capture / Keylogging
T1113 · T1056.001
▼
📎 INSERT RULE LINK 🔗 T1113 Screen Capture 🔗 T1056.001 Keylogging
⚡ KILL CHAIN: Collection (credential harvest, intelligence, long-term dwell)
🔵 BLUE TEAM
Screen capture and keylogging are indicators of long-term dwell and intelligence collection — characteristic of espionage actors. Detection: processes calling BitBlt or CopyImage GDI functions at regular intervals (screenshot polling), processes installing keyboard hooks via SetWindowsHookEx with WH_KEYBOARD_LL — especially from non-accessibility software processes. Periodic small file creation in TEMP at regular intervals (screenshot staging) is a behavioural indicator.
🔴 RED TEAM
Used by RATs for intelligence collection during extended dwell periods. Keyloggers harvest credentials entered after the initial compromise — catching VPN credentials, banking credentials, new passwords set after a suspected incident. Screen capture confirms the attacker is watching the right target before deploying final payload.
