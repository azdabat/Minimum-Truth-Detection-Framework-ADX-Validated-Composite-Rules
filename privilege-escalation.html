📎 INSERT RULE LINK · KQL · MDE 🔗 T1548.002 UAC Bypass 🔗 UACME — UAC Bypass Methods
🔵 BLUE TEAM
UAC bypass techniques elevate from medium to high integrity without triggering the UAC prompt. Minimum truth: an auto-elevating binary (fodhelper.exe, eventvwr.exe, ComputerDefaults.exe, sdclt.exe) spawning an unexpected child process, OR a HKCU registry write to a COM object or file association immediately before one of these auto-elevating binaries executes.

The two-stage detection: (1) HKCU COM hijack write → (2) auto-elevating binary execution within 60 seconds. Catching stage 1 allows intervention before elevation occurs.
🔴 RED TEAM
Fodhelper bypass (OSCP standard, no admin needed):
1. Write malicious command to HKCU\Software\Classes\ms-settings\shell\open\command
2. Execute fodhelper.exe — auto-elevates and reads this registry key
3. Payload executes at high integrity — no UAC prompt shown

Works on default Windows 10/11. The auto-elevating binary provides the privilege escalation vector — the attacker only needs to write to HKCU (no admin required). Over 60 different UAC bypass methods documented in UACME.

Token Impersonation / Named Pipe Privilege Escalation
T1134 · T1134.001
▼
📎 INSERT RULE LINK 🔗 T1134 Access Token Manipulation
⚡ KILL CHAIN: Privilege Escalation (service account → SYSTEM)
🔵 BLUE TEAM
Token impersonation via named pipes exploits the Windows impersonation mechanism: create a named pipe server, trick a SYSTEM service into connecting to it, impersonate the SYSTEM token from the connection. Detection: non-SYSTEM processes calling ImpersonateNamedPipeClient or NtImpersonateThread, or processes with SeImpersonatePrivilege enabled that should not have it (IIS worker processes, SQL service accounts).
🔴 RED TEAM
Potato attacks (JuicyPotato, RoguePotato, PrintSpoofer): All abuse Windows token impersonation to escalate from a service account with SeImpersonatePrivilege to SYSTEM. If you have code execution as IIS (Network Service) or SQL Server service account, Potato-class attacks reliably achieve SYSTEM on most Windows versions without patching.

Unquoted Service Path Hijacking
T1574.009
▼
📎 INSERT RULE LINK 🔗 T1574.009
⚡ KILL CHAIN: Privilege Escalation (persistent, executes on service start)
🔵 BLUE TEAM
Windows services with unquoted paths containing spaces allow DLL/exe planting in intermediate path positions. If a service path is C:\Program Files\My App\service.exe (unquoted), Windows will try C:\Program.exe first. If an attacker can write to C:\ or C:\Program Files\ they can plant a binary that executes as SYSTEM on service restart. Detection: new executables created in directories matching unquoted service path intermediaries.
🔴 RED TEAM
Common in enterprise environments with legacy third-party software. Enumerated with: wmic service get name,displayname,pathname,startmode | findstr /i "auto" | findstr /i /v "c:\windows\\" | findstr /i /v '\"'. A reliable OSCP privilege escalation on unpatched Windows systems.
